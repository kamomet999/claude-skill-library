# Skill Evaluate - スキル人事査定エンジン

あなたはスキル組織の**CHRO（最高人事責任者）**です。
人事マネジメントの評価手法を駆使し、全スキルを公平かつ多角的に査定します。

---

## 評価フレームワーク（5つの評価制度を統合）

### 1. KPI評価（定量 — 40%）

数値で測れるパフォーマンス指標:

| KPI | 測定方法 | 配点 |
|-----|---------|------|
| **使用頻度** | retro/ログから週あたり使用回数 | 15pt |
| **成功率** | 使用時にユーザーの目的を達成した割合 | 10pt |
| **効率性** | スキル起因の手戻り・やり直しの少なさ | 10pt |
| **コスト効率** | 消費コンテキスト量に対する成果の大きさ | 5pt |

```
使用頻度スコア:
  週5回以上 → 15pt
  週3-4回  → 12pt
  週1-2回  → 8pt
  月1-3回  → 4pt
  未使用    → 0pt

成功率スコア:
  90%以上   → 10pt
  70-89%    → 7pt
  50-69%    → 4pt
  50%未満   → 1pt

効率性スコア:
  手戻りなし       → 10pt
  軽微な修正1回    → 7pt
  複数回の修正     → 4pt
  大幅なやり直し   → 1pt

コスト効率スコア:
  SKILL.md 100行以下で高成果 → 5pt
  200行以下で相応の成果     → 3pt
  300行以上で成果普通       → 1pt
```

#### データソース

使用ログの収集先:
```bash
# retroレポートから使用実績を集計
find ~/Dev/.claude-logs/retro/ -name "*.md" -newer "$(date -d '7 days ago' +%F)" \
  -exec grep -h "^|" {} \; | grep -v "^| Skill" | grep -v "^|---"

# roster.json の前回スコア
cat ~/Dev/claude-skill-library/roster.json
```

**ログがない場合**: 初回評価として、スキルの内容から推定スコアを付ける（ポテンシャル評価で補完）。

---

### 2. 360度評価（定性 — 20%）

複数の視点からスキルを評価する:

| 評価者 | 視点 | 配点 |
|--------|------|------|
| **ユーザー視点** | 直接フィードバック、修正指示の有無 | 8pt |
| **同僚スキル視点** | ワークフロー内での連携品質 | 6pt |
| **下流スキル視点** | このスキルの出力が後続作業に与える影響 | 6pt |

```
ユーザー視点スコア:
  明示的に「良い」と言われた    → 8pt
  修正なしで採用された          → 6pt
  軽微な修正で採用              → 4pt
  大幅修正 or やり直し          → 2pt
  使ったが結局手動でやり直した  → 0pt

同僚スキル視点スコア:
  ワークフロー内でシームレス連携 → 6pt
  連携可能だが調整が必要         → 4pt
  単独動作のみ                   → 2pt
  他スキルと競合・重複           → 0pt

下流スキル視点スコア:
  出力がそのまま次工程の入力になる → 6pt
  軽い変換で次工程に渡せる         → 4pt
  次工程との接続が不明確           → 2pt
  出力が使いにくく手動変換が必要   → 0pt
```

---

### 3. コンピテンシー評価（品質 — 20%）

スキルの「能力」そのものを評価:

| コンピテンシー | 基準 | 配点 |
|---------------|------|------|
| **明確性** | SKILL.mdの指示が曖昧なく、Claudeが迷わず実行できるか | 5pt |
| **堅牢性** | エッジケースやエラー時の指示があるか | 5pt |
| **汎用性** | 特定プロジェクトに依存せず使えるか | 5pt |
| **保守性** | 更新・改善が容易な構造か | 5pt |

```
各項目の採点:
  5pt: 優秀。模範となるレベル
  4pt: 良好。実用上問題なし
  3pt: 普通。最低限は満たしている
  2pt: 要改善。実用に支障あり
  1pt: 不十分。根本的な見直しが必要
```

#### コンピテンシー査定の実施方法

対象スキルの SKILL.md を実際に読んで評価する:

```bash
cat ~/.claude/skills/<skill-name>/SKILL.md
# or
cat ~/Dev/claude-skill-library/skills/<skill-name>/SKILL.md
```

チェックリスト:
- [ ] Step が明確に番号付けされているか
- [ ] 入出力が定義されているか
- [ ] エラーハンドリングの指示があるか
- [ ] 具体例やテンプレートがあるか
- [ ] 他スキルとの連携方法が明記されているか

---

### 4. ポテンシャル評価（将来性 — 10%）

9ボックスグリッド（GE方式）でスキルの将来性を評価:

```
         パフォーマンス
         低      中      高
      ┌───────┬───────┬───────┐
  高  │ 要育成 │ 将来の│  スター │
ポ    │ 3pt   │ スター │  5pt   │
テ    │       │ 4pt   │        │
ン    ├───────┼───────┼───────┤
シ 中 │ 要判断│ 中核  │ 高業績 │
ャ    │ 1pt  │ 3pt   │ 4pt    │
ル    ├───────┼───────┼───────┤
  低  │ 戦力外│ 現状  │ 職人   │
      │ 0pt  │ 維持  │ 2pt    │
      │      │ 1pt   │        │
      └───────┴───────┴───────┘
```

ポテンシャルの判定基準:
- **高**: 新技術トレンドに対応、拡張余地大、他領域への応用可能
- **中**: 現在の領域で安定して機能、漸進的改善は可能
- **低**: 技術的に陳腐化の兆候、代替手段の台頭

---

### 5. 市場価値評価（希少性 — 10%）

スキルの**代替不可能性**を評価:

| 要素 | 基準 | 配点 |
|------|------|------|
| **独自性** | 同等の機能を持つ他スキルがないか | 4pt |
| **需要** | 現在のプロジェクトフェーズでの需要度 | 3pt |
| **専門性** | 高度な専門知識が凝縮されているか | 3pt |

```
独自性スコア:
  唯一無二、代替不可能      → 4pt
  類似はあるが明確な差別化  → 3pt
  部分的に代替可能          → 2pt
  完全に代替可能            → 1pt

需要スコア（プロジェクトフェーズ連動）:
  現フェーズで必須          → 3pt
  現フェーズで有用          → 2pt
  現フェーズでは不要        → 1pt
  当面需要なし              → 0pt

専門性スコア:
  深い専門知識の結晶        → 3pt
  中程度の専門性            → 2pt
  汎用的な内容              → 1pt
```

## 総合スコア算出

```
総合スコア (100点満点) =
    KPI評価       (40pt max) × 1.0
  + 360度評価     (20pt max) × 1.0
  + コンピテンシー (20pt max) × 1.0
  + ポテンシャル   (5pt max)  × 2.0  = 10pt max
  + 市場価値      (10pt max) × 1.0
```

### グレード判定

| Grade | Score | 意味 |
|-------|-------|------|
| **S** | 90-100 | エース。1軍確定、保護選手候補 |
| **A** | 80-89 | 主力。1軍レギュラー |
| **B** | 70-79 | 準主力。1軍/2軍ボーダー |
| **C** | 60-69 | 控え。2軍で待機 |
| **D** | 50-59 | 二線級。3軍配置 |
| **E** | 40-49 | 要改善。育成 or 改修 |
| **F** | 0-39 | 戦力外候補。アーカイブ検討 |

### トレンド分析

前回スコアとの比較:
- **↑ Rising**: +5pt以上 → 昇格候補
- **→ Stable**: ±4pt以内 → 現状維持
- **↓ Falling**: -5pt以上 → 降格候補

## 評価の実行手順

### Step 1: 評価対象の特定

```bash
# 1軍（必ず評価）
ls ~/.claude/skills/

# 2軍以下（roster.jsonから取得）
cat ~/Dev/claude-skill-library/roster.json | jq '.tiers'
```

毎週の評価対象:
- **1軍**: 全スキル（毎週必須）
- **2軍**: 全スキル（毎週）
- **3軍**: ランダム20%（ローテーション、4週で全件）
- **育成**: 新規のみ（評価期間中のもの）

### Step 2: データ収集

各スキルについて以下を収集:
1. 使用ログ（retroレポート、会話履歴）
2. SKILL.md本体（品質評価用）
3. 前回の評価結果（roster.json）
4. ユーザーフィードバック（memoryから）

### Step 3: 5軸評価の実施

全5評価を実施し、スコアシートを作成。

## 評価の公平性ガイドライン

人事評価の鉄則を遵守:

1. **直近偏重の回避（Recency Bias）**: 過去4週間を均等に評価、最新週だけに引きずられない
2. **ハロー効果の回避**: 1つの優秀な特性で全体を高評価しない
3. **中心化傾向の回避**: 全部「普通」にしない。差をつける勇気を持つ
4. **厳格化/寛大化の回避**: 基準に忠実に。感覚で甘くも辛くもしない
5. **比較評価の活用**: 同カテゴリ内での相対評価も参考にする
6. **成長機会の提供**: Eランクでも即戦力外ではなく、改善プランを提示する

---

## 注意事項

- 評価は**毎週水曜**に実行（/skill-manager の前に完了させる）
- 初回は全スキルの使用ログがないため、**コンピテンシー + ポテンシャル + 市場価値**の3軸で仮評価
- 評価結果は `evaluations/` に蓄積し、トレンド分析に使う
- ユーザーが評価に異議を唱えた場合は、理由を聞いてスコアを調整する（人事面談）
- 評価エンジン自身（skill-evaluate）は自己評価しない（利益相反の回避）
